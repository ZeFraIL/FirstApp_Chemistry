# Описание класса: MainActivity

## 1. Общая информация
*   **Имя класса:** `MainActivity`
*   **Тип:** Activity (Активность/Экран)
*   **Назначение:** Это основной экран приложения. Он выполняет роль «контроллера», который управляет логикой выбора химических соединений и проверки их состава.
*   **Взаимодействие:** Взаимодействует с системой Android для отображения интерфейса, загружает данные из ресурсов проекта (`strings.xml`) и обрабатывает ввод пользователя через списки и кнопки.

---

## 2. Переменные (Поля класса)

| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `lvElements` | `ListView` | Список для отображения доступных химических элементов. | `onCreate` |
| `lvCompounds` | `ListView` | Список для отображения химических соединений. | `onCreate` |
| `tvSelectedCompound` | `TextView` | Текстовое поле, показывающее выбранное соединение. | `onItemClick`, `resetSelection` |
| `tvUserSelectedElements` | `TextView` | Поле, отображающее элементы, которые выбрал пользователь. | `updateUserSelectedElementsTextView` |
| `btnReady` | `Button` | Кнопка «Готово» для запуска проверки результата. | `onCreate` |
| `elementsArray` | `String[]` | Массив строк с названиями элементов из ресурсов. | `onCreate`, `onItemClick` |
| `compoundsArray` | `String[]` | Массив строк с названиями соединений из ресурсов. | `onCreate`, `onItemClick` |
| `currentSelectedCompoundFormula` | `String` | Хранит формулу выбранного соединения (например, «H2O»). | Во всем классе для отслеживания состояния. |
| `requiredElementsForCompound` | `List<String>` | Список всех атомов, необходимых для формулы. | `parseFormula`, `btnReady.onClick` |

## 3. Методы класса

### Метод: `onCreate`
*   **Тип:** `protected`
*   **Возвращаемое значение:** `void` (ничего не возвращает)
*   **Параметры:**
    | Имя | Тип | Описание |
    | :--- | :--- | :--- |
    | `savedInstanceState` | `Bundle` | Объект с сохраненным состоянием экрана. |
*   **Что делает:**
    1. Устанавливает внешний вид экрана через `setContentView`.
    2. Связывает Java-переменные с элементами интерфейса (кнопки, списки) через `findViewById`.
    3. Загружает данные о химикатах из ресурсов приложения.
    4. Настраивает «Адаптеры», которые объясняют спискам, как отображать данные.
    5. Устанавливает «Слушатели» (код, который срабатывает при нажатии).
*   **Когда вызывается:** Автоматически системой Android при запуске приложения.

### Метод: `updateUserSelectedElementsTextView`
*   **Тип:** `private`
*   **Что делает:** 
    1. Проверяет, пуст ли список выбранных пользователем элементов.
    2. Если не пуст, объединяет их в одну строку (например, «H, H, O») и выводит на экран.
*   **Когда вызывается:** При каждом добавлении элемента пользователем или сбросе выбора.

### Метод: `parseFormula`
*   **Тип:** `private`
*   **Возвращаемое значение:** `List<String>` (Список символов элементов)
*   **Параметры:**
    | Имя | Тип | Описание |
    | :--- | :--- | :--- |
    | `formula` | `String` | Формула, которую нужно разобрать (например, «H2O»). |
*   **Что делает (подробно):**
    1. **Нормализация:** Заменяет спецсимволы подстрочных цифр (₂ или ₃) на обычные (2 или 3).
    2. **Поиск паттернов:** Использует **Регулярные выражения** (`[A-Z][a-z]?`), чтобы найти символы элементов, начинающиеся с заглавной буквы.
    3. **Подсчет:** Ищет цифры после символа. Если цифра есть (например, '2'), добавляет этот элемент в список нужное количество раз.
*   **Когда вызывается:** Когда пользователь выбирает соединение из списка.
*   **Важно:** Это «мозг» приложения. Он превращает строку «CO2» в список: `["C", "O", "O"]`.

## 4. Жизненный цикл

*   **`onCreate()`**:
    *   **Когда вызывается:** При создании активности.
    *   **Действие:** Инициализация всех элементов. Это этап «подготовки» экрана к работе.

---

## 5. Взаимодействие с интерфейсом (UI)
*   **Элементы:** `ListView` (списки), `Button` (кнопка), `TextView` (текст).
*   **Связь:** Код использует `findViewById(R.id...)` для соединения XML-дизайна с объектами Java.
*   **События:** 
    *   `onItemClick`: нажатие на элемент списка.
    *   `onClick`: нажатие на кнопку «Готово».

---

## 6. Взаимодействие с другими компонентами
*   **Ресурсы:** Класс активно использует `strings.xml` для получения данных.
*   **Уведомления:** Использует `Toast` для быстрого показа сообщений («Верно!» / «Ошибка»).

---

## 7. Общая логика класса
Приложение работает по шагам:
1.  Пользователь выбирает **Соединение**.
2.  Программа вычисляет, какие **Элементы** нужны (цель).
3.  Пользователь набирает элементы по одному.
4.  Программа сравнивает набор пользователя с целью (предварительно отсортировав списки, чтобы порядок не имел значения).
5.  Если списки совпали — победа!

---

## 8. Упрощенное объяснение
**Аналогия: Конструктор Lego**
Представьте, что `MainActivity` — это **инструкция к Lego**.
1.  Сначала вы выбираете, какую **модель** хотите собрать (Соединение).
2.  Инструкция говорит вам, какие именно **детали** нужны (логика `parseFormula`).
3.  Вы начинаете выбирать детали из общей коробки (список Элементов).
4.  Когда решили, что закончили, показываете учителю (кнопка «Готово»).
5.  Учитель проверяет количество деталей. Если всё верно — вы получаете наклейку «Молодец!».

## 🛠 Возможные улучшения
*   **Скобки:** Метод `parseFormula` не умеет работать со сложными формулами со скобками, например `(NH4)2SO4`.
*   **Кнопка отмены:** Если пользователь ошибся, ему приходится заново выбирать соединение. Было бы полезно добавить кнопку «Удалить последний элемент».

## 🎯 Цель проекта
*   **Решаемая задача:** Приложение помогает студентам и любителям химии изучать и запоминать химический состав различных соединений.
*   **Важность:** Изучение химии может быть сложным; интерактивный способ «сборки» соединений из элементов делает процесс более увлекательным и запоминающимся.
*   **Целевая аудитория:** Студенты, школьники и все, кто интересуется основами химии.

---

## 📌 Требования к приложению
### Функциональные требования
*   Отображение списка распространенных химических соединений.
*   Отображение списка химических элементов.
*   Возможность выбрать соединение, а затем «собрать» его, выбирая элементы.
*   Проверка соответствия выбора пользователя химической формуле.
*   Обратная связь через Toast-сообщения (Верно/Неверно).

### Нефункциональные требования
*   **Производительность:** Мгновенный отклик на действия пользователя и парсинг формул.
*   **Удобство использования:** Простой одноэкранный интерфейс с понятными списками.
*   **Надёжность:** Надежный парсинг формул с использованием регулярных выражений (Regex).

---

## 🧠 Общая архитектура
*   **Подход:** MVC (Model-View-Controller) / Простая архитектура на основе Activity.
*   **Почему выбран именно он:** Учитывая простую природу проекта (один экран, локальные данные), стандартный подход, ориентированный на Activity, является эффективным и простым в поддержке без лишнего усложнения кода.
*   **Основные компоненты системы:**
    *   `MainActivity`: Управляет логикой пользовательского интерфейса и взаимодействием.
    *   `strings.xml`: Служит источником данных (массивы элементов и соединений).
    *   `parseFormula()`: Внутренняя логика разложения химических формул.

## 🧩 Подробное описание классов
### 📌 Класс: MainActivity
*   **Роль:** Основной контроллер и представление приложения.
*   **Ответственность:** Инициализация UI, загрузка данных из ресурсов, обработка нажатий на списки и валидация формул.
*   **Основные методы:**
    *   `onCreate()`: Настройка макета, адаптеров и слушателей.
    *   `parseFormula(String formula)`: Использует Regex для извлечения элементов и их количества из строки (например, «H₂O»).
    *   `updateUserSelectedElementsTextView()`: Обновляет UI для отображения текущего выбора.
*   **Взаимодействие:** Взаимодействует с адаптерами `ListView` и использует стандартные Android Toast для уведомлений.

---

## 🔄 Диаграмма работы приложения (Сценарий)
1.  Пользователь выбирает соединение (например, Вода — H₂O).
2.  Приложение парсит формулу и очищает предыдущий выбор элементов.
3.  Пользователь нажимает на элементы (например, H, H, O) в списке элементов.
4.  Пользователь нажимает кнопку «Ready».
5.  Приложение сравнивает отсортированный список необходимых элементов с выбором пользователя и выводит результат.

## 🎨 UI/UX анализ
*   **Почему интерфейс сделан именно так:** Разделение экрана на два списка для удобного сравнения и выбора.
*   **Использованные принципы:**
    *   **Простота:** Нет скрытых меню; все на одном экране.
    *   **Логичность:** Поток действий сверху вниз (Выбор соединения -> Выбор элементов -> Проверка).
    *   **Доступность:** Крупный текст в элементах списка для лучшей читаемости.
*   **Что можно улучшить:** Добавить изображения соединений или визуализацию периодической таблицы.

---

## ⚙️ Работа с потоками
*   **Используются:** UI Thread (Основной поток).
*   **Почему выбран этот способ:** Все операции (парсинг коротких строк) выполняются чрезвычайно быстро и не блокируют интерфейс.
*   **Предотвращение зависаний (ANR):** Логика максимально облегчена.

---

## 💾 Работа с данными
*   **Где хранятся данные:** Файлы ресурсов Android (`strings.xml`).
*   **Почему выбран этот способ:** Статические данные лучше всего хранить в ресурсах для поддержки локализации и простоты.
*   **Сохранность:** Данные доступны только для чтения во время выполнения, что гарантирует их целостность.

---

## 🧪 Тестирование
*   **Ручное тестирование:** Проверка всех соединений в списке.
*   **Валидация:** Тестирование парсера Regex с различными формулами (индексы, одно- и двухбуквенные символы).

---

## 🐞 Обработка ошибок
*   **Предусмотренные ошибки:**
    *   Нажатие «Ready» без выбора соединения.
    *   Выбор элементов без предварительного выбора соединения.
*   **Реакция:** Информативные Toast-сообщения, направляющие пользователя.

---

## ⚡ Производительность
*   **Оптимизации:** Использование `ArrayAdapter` с кастомными макетами для эффективного отображения списков.
*   **Узкие места:** Не ожидаются при текущем объеме данных.
